Light/Dark Mode — CSS Pattern
css
/* System preference is the default. Manual toggle overrides via data-theme attribute. */
:root, [data-theme="dark"]  { /* dark tokens */ }
[data-theme="light"]         { /* light tokens */ }

@media (prefers-color-scheme: dark) {
  :root:not([data-theme]) { /* dark tokens — system fallback */ }
}
Typography

Components

Buttons

One primary action per view. Solid fill only — never gradient buttons. Secondary uses surface elevation. Ghost for low-hierarchy actions. Icon buttons always need aria-label.

Quality Control

Anti-patterns

If it looks like it could have been generated by any AI tool, it fails. These patterns instantly signal template generation. Avoid unconditionally.

Anti-patternWhy it failsVOID alternative
Gradient buttonsScreams template / AI slopSolid --color-primary fill only
Purple orbs / neon blobsOverused AI aesthetic markerSingle radial at 15% opacity max
Colored side-border cardsMaterial Design 1.0 tellSurface elevation + hairline alpha border
Icons in colored circlesSaaS starter template lookIcon alone, no background shape
Centering everythingAI default; creates no hierarchyLeft-align body; center hero headings only
3-up symmetrical feature gridMost recognizable AI layout patternAsymmetric 2+1, staggered, varying heights
Generic hero copy"Unlock the power of…" = instant AI tellSpecific claim unique to this product
Multiple bright accent colorsLoses hierarchy, toy-like appearance1 accent + neutrals; more only for charts
Quick reference — accessibility checklist
✓ Semantic HTML: <header> <nav> <main> <section> <article> <footer>
✓ One <h1> per page — heading hierarchy never skips levels
✓ WCAG AA: 4.5:1 body text, 3:1 large text (24px+)
✓ Keyboard: Tab/Enter/Space/Escape for all interactive elements
✓ Alt text on every <img> — decorative: alt=""
✓ Touch targets 44×44px minimum
✓ prefers-reduced-motion: disable all animations
✓ aria-label on every icon-only button
✓ Skip link as first focusable element
✓ No hover-dependent UI on mobile
