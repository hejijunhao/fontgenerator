### 0.2 â€” Preprocess

### 0.3 â€” Trace (Potrace)

```bash

```

**Note on `alphamax`:** In Potrace, *lower* values preserve sharper corners; *higher*

### 0.4 â€” Place in em-space + winding

**Font coordinate system:**

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| `unitsPerEm` | 1000 | Blueprint default |
| `canvasHeight` H | 1254 | Full source height (shared frame) |
| `baselineFraction` b | 946/1254 â‰ˆ **0.754385** | Foot row (bottom ink) â†’ font `y=0` |
| scale s | U/H â‰ˆ **0.797448** | px â†’ font units |

**Transform discovery (important for Phase 2):**

FontForge's `importOutlines()` **already applies** the Potrace SVG transform group
(`translate(0,H) scale(0.1,-0.1)`). Imported coordinates are **y-up** in SVG viewBox
space â€” not raw y-down image rows.

Therefore the build script does **not** apply a Y-flip. Instead:

```python
glyph.correctDirection()  # mandatory â€” punches counter hole in CFF
_xmin, ymin, _xmax, _ymax = glyph.boundingBox()
glyph.transform((s, 0, 0, s, leftBearing, -ymin * s))  # foot â†’ y=0, scale to em
```

Initial attempt used `sy=-s, dy=b*U` (naÃ¯ve y-downâ†’y-up flip). That left the glyph
floating at `yminâ‰ˆ268`. Corrected transform anchors foot at `ymin=0`.

**Final glyph metrics:**

| Metric | Value (font units) |
|--------|-------------------|
| advance width | 654 |
| cap height (bbox) | ~451 |
| left bearing | 40 |
| right bearing | 40 |
| bbox | (40.3, 0.0, 497.4, 450.8) |

`capHeight` (~451) < `bÃ—U` (754) because the letter doesn't fill the full em ascender â€”
expected for this capture frame.

## Handoff to Phase 2

### Pinned defaults (`scripts/cli-reference/params.json`)

Use these as hardcoded no-agent defaults in the browser pipeline:

```json
{
  "preprocess": { "threshold": 0.6, "close": 1, "invert": false },
  "trace": { "turdsize": 2, "alphamax": 1.0, "opttolerance": 0.2 },
  "place": { "unitsPerEm": 1000, "baselineFraction": 0.754385 },
  "metrics": { "leftBearing": 40, "rightBearing": 40 }
}
```

### pxâ†’em math to port (TypeScript)

For a source pixel row `r` (0 = top of **full** canvas) and column `x` (after
horizontal crop, offset by `inkLeft`):

```
s = unitsPerEm / canvasHeight
baselineRow = baselineFraction * canvasHeight
fontX = (x + inkLeft - inkLeft) * s + leftBearing   # cropped x + bearings
fontY = (baselineRow - r) * s                         # y-up, baseline at 0
```

When importing SVG paths from Potrace-WASM instead of FontForge, apply the equivalent
transform on path coordinates. Run `correctDirection` / winding fix **after** import.

### Master format decision (Phase 2 fork)

CLI master is **CFF OTF** (FontForge default). opentype.js CFF writing is limited â€”
Phase 2 should spike early whether to:

1. Emit **TTF/glyf** as browser master (likely), or
2. Match CLI CFF via fonttools WASM

Golden OTF remains the correctness reference regardless.

### Surprises / lessons

1. **FontForge SVG import is not naive y-down pixels** â€” always inspect `boundingBox()`
   after import before applying transforms.
2. **`correctDirection()` is mandatory** for `A`; without it CFF non-zero fill would
   paint the counter solid.
3. **Threshold 60%** works well on cream `#f1efe2` background without alpha.
4. Potrace emits one compound path for outer+counter â€” winding fix operates on
   subpaths.
